# 📡 03. 내장 센서 스위트 (Sensor Suite & Fusion)

> "VLM이 두뇌라면, 센서는 눈·귀·코다. 그물은 한 번에 한 발이라, **표적을 놓치면 ㅈ된다.** 그래서 센서는 ㅈㄴ 중요함."

---

## 0. 센서 설계 원칙

1. **탐지(Detect) → 추적(Track) → 조준(Aim)** 전 과정 끊김 없이.
2. **수동(passive) 우선.** 적이 우리 레이더 전파 보고 도망가면 안 됨 → EO/IR/음향 위주, 레이더는 보조.
3. **악조건 내성.** 야간·역광·안개·재밍에서도 동작.
4. **거리 정밀도.** 그물 사거리(수십 m)에 정확히 들어왔는지 cm급으로 알아야 발사함.

---

## 1. 센서 구성표 (Sensor Stack)

| # | 센서 | 역할 | 비고 |
|---|------|------|------|
| 1 | **EO 카메라 (광학 줌)** | 원거리 탐지/식별 | 30x 광학줌, 4K |
| 2 | **IR/열화상 카메라** | 야간/연막/엔진열 탐지 | 자폭드론 배터리·기폭부 열징후 |
| 3 | **소형 AESA 레이더** (보조) | 전방위 1차 탐지, 거리/속도 | 저출력, 필요시만 ON |
| 4 | **음향 어레이 (마이크 배열)** | 프로펠러 소리로 방위 탐지 | 완전 수동, 재밍 무관 |
| 5 | **RF 패시브 탐지** | 적 드론 통신/영상 송신 전파 청취 | 송신만 들음(스푸핑 X) |
| 6 | **LiDAR / 거리계** | 근접 조준 시 cm급 거리 | 그물 발사 타이밍 결정 |
| 7 | **RTK GPS + IMU** | 자기 위치/자세 | 호버 조준 정밀도 |
| 8 | **ADS-B 수신기** | 민항기/아군기 회피 | 안전 |

## 2. 센서별 상세

### 2.1 EO 카메라 (광학)
- 4K, 30x 광학줌 + 디지털 안정화 짐벌(3축)
- 1.5 km 거리에서 드론 식별 가능 목표
- HawkEye-VLM의 주 입력 (형상/페이로드 식별)

### 2.2 IR / 열화상
- 장파장 열화상(LWIR), 640x512급
- **자폭드론 탐지 비밀병기:** 배터리/모터/기폭회로의 열징후로 "이거 뭔가 있다" 감지
- 야간·연막·역광에서도 동작 → 24시간 근무

### 2.3 소형 AESA 레이더 (보조·수동 우선 정책)
- 평소엔 꺼두고, 음향/RF로 1차 포착 후 **정밀 거리/속도 측정 필요할 때만 잠깐 ON**
- 이유: 항상 켜두면 적이 우리 레이더파 보고 "어 저기 사냥꾼 있네" 하고 튐

### 2.4 음향 어레이 (Acoustic Array)
- 동체 둘레에 MEMS 마이크 배열
- 프로펠러 특유의 주파수(BPF)로 방위각 추정
- **완전 수동** → 재밍/스텔스 환경에서도 "윙~" 소리만 나면 잡아냄
- 소형·저속 드론 조기경보에 강함

### 2.5 RF 패시브 탐지 (중요: 듣기만 함)
- 적 드론↔조종기, 적 드론↔영상송신 전파를 **수신만** 함
- ❗ **스푸핑/재밍 안 함** (그게 우리 윤리 셀링포인트) → "엿듣되 손대지 않는다"
- 신호 특성으로 드론 모델/제조사 추정 가능

### 2.6 LiDAR / 근접 거리계
- 표적이 그물 사거리에 들어왔는지 cm급 판정 → **발사 트리거**
- 너무 멀면 빗나가고, 너무 가까우면 충돌. 골디락스 존 계산.

## 3. 센서 융합 파이프라인 (Sensor Fusion)

```
 [음향]──┐
 [RF]────┤ 1차 탐지/조기경보 ─▶ "12시 방향에 뭔가 있다"
 [레이더]┘                         │
                                    ▼
 [EO]────┐ 2차 식별/추적 ─▶ HawkEye-VLM ─▶ 적/위협/결심
 [IR]────┘                         │
                                    ▼
 [LiDAR]─┐ 3차 정밀조준 ─▶ 사거리 진입 판정 ─▶ 🪤 그물 발사!
 [RTK]───┘
                                    ▼
                          멀티표적 트래커(MOT)
                       (칼만필터/JPDA로 다표적 동시 추적)
```

| 단계 | 사용 센서 | 산출물 |
|------|-----------|--------|
| 조기경보 | 음향 + RF (+레이더) | 방위/대략거리 |
| 식별·분류 | EO + IR + VLM | 클래스/위협/결심 |
| 정밀조준 | LiDAR + RTK + IMU | 발사 솔루션(거리·리드각) |
| 다표적 관리 | 전체 융합 | 트랙 ID, 우선순위 큐 |

> 군집(swarm)이 몰려오면? **다중객체추적(MOT)** 으로 ID 붙이고 위협순위 매겨서 "제일 ㅈ될 것 같은 놈부터" 처리.

---

## 4. 악조건 대응 (Adverse Conditions)

| 조건 | 대응 센서/전략 |
|------|----------------|
| 야간 | IR 주도 + 저조도 EO |
| 안개/연막 | IR + 레이더 + 음향 |
| 역광(태양) | IR + 편광필터 EO |
| 재밍 환경 | 음향 + IR (완전 수동, GPS 끊겨도 비전측위) |
| 스텔스 소형기 | 음향 어레이 (소리는 못 숨김) |

---

## 5. 데이터 인터페이스

- 모든 센서 → ROS 2 토픽으로 시간동기화(PTP) 퍼블리시
- VLM은 EO/IR 프레임 + 센서 메타데이터 토큰을 입력으로 받음
- 블랙박스: 전 센서 raw + 융합결과 NVMe에 로깅

---

## 6. 센서 스펙 요약

```yaml
eo_camera: 4K, 30x optical zoom, 3-axis gimbal
ir_camera: LWIR 640x512
radar: compact AESA (low-power, on-demand)
acoustic: MEMS mic array (passive, 360°)
rf_detect: passive receive-only (no spoof/jam)
rangefinder: LiDAR, cm-grade at net range
positioning: RTK-GPS + triple IMU
safety: ADS-B receiver
fusion: ROS2 + Kalman/JPDA multi-object tracking
detection_range_target_km: 1.5 (EO ID), 3+ (acoustic/RF early warning)
```
