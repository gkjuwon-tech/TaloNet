# 12. 차고 수제작 가이드 — "그물매 & ForensIQ-1 손으로 만들기"

> **부제: 방산기업이 없어서 차고에서 시작한다 (feat. 옆집 김씨의 항의)**
>
> 이 문서는 [BOM(`bom/TaloNet_BOM.xlsx`)](../bom/TaloNet_BOM.xlsx)대로 부품 사서
> **CNC 같은 건 외주 발품 팔고, 나머지는 차고에서 직접** 두 개의 완성품
> — **그물 단 드론(그물매)** 과 **포렌식 기기(ForensIQ-1)** — 을 만들어
> **실제로 소프트웨어(`gcs/`, `forensics/`)랑 통신되고 스펙 목표치 다 돌아가게** 하는
> 전 과정이다. 진지한 건 진지하게, 웃긴 건 웃기게. 둘 다 안 지키면 손가락 날아간다.

---

## 0. TL;DR (바쁜 차고 장인용)

1. **BOM 보고 발주** → 외주(카본/PCB)는 리드타임 길어서 **제일 먼저** 쏜다.
2. **드론**: 프레임 → 추진 → 항전 → **그물 페이로드(조준짐벌/시닝/윈치/트리거)** → 배선 → 펌웨어.
3. **포렌식 기기**: 케이스 프린트 → 전장 조립 → 소프트웨어 굽기.
4. **브링업**: `payload_map.py`의 채널/각도 = 실제 서보 = `SERVOn_MIN/MAX/TRIM` 일치시킴.
5. **검증**: `python -m gcs.sim_vehicle` ↔ `python -m gcs --connect` 로 **실제 명령/텔레메트리** 확인 → 실 FC → 벤치 → 비행.

> 한 줄: **"찰흙(SCAD)이 쇠(실물)가 되고, 키보드가 그물을 쏜다."**

## 1. 차고 셋업 & 안전 (이거 안 읽으면 진짜 죽음)

차고에 들이는 순간 당신 차고는 **항공우주국 + 화약고 + 전자공방**이 된다. 공구는 BOM
`Shop Tools & Equipment` 시트 참조. 안전은 농담 아님:

- 🔋 **LiPo/Li-ion 12S(50V급)**: 단락 = 작은 태양. **LiPo-safe 백/벙커**에 보관·충전,
  쇼트 방지, 풀충전 채로 방치 금지. 부풀면(임신 배터리) **즉시 격리·폐기**. 차고에 소화기(D급/모래).
- 💨 **CO2 솔레노이드/카트리지(발사부)**: 고압. 카트리지 정면에 얼굴 두지 마라.
  **그물 발사 테스트는 실내 금지** — 마당이나 빈 공간에서, **사람·창문·옆집 김씨 강아지 없는 방향**으로.
  처음엔 그물 빼고 **빈 발사(dry-fire)** 로 트리거만 확인.
- 🔥 **솔더링/핫에어**: 흄 익스트랙터 필수(납연기 = 머리 나빠짐, 안 그래도 이거 만들 정도면…).
- ✂️ **프롭 28인치 8개**: 돌면 **손가락 절단기**. 추진계 테스트는 **프롭 빼고**, 모터만. 프롭 달면 묶어둠.
- 🧤 보안경 + 장갑 + 환기. **혼자 작업 금지**(2인 원칙 = 무결성 + 응급).

> ⚠️ **법/윤리:** 이건 **인가된 운용자**용 대드론 시스템이다. 시스템은 **조종/조준/정보 + 사람이
> 당기는 방아쇠**까지 — 자율 무기 아님. 실제 RF/공역/무기류 규제는 당신 관할 법을 따른다.
> 차고에서 만드는 건 자유지만 **하늘에 띄우고 쏘는 건 다른 얘기**. 변호사 친구 하나 만들어둬라.

## 2. 발주 — BOM 보고 장바구니 (리드타임 게임)

[`bom/TaloNet_BOM.xlsx`](../bom/TaloNet_BOM.xlsx) 열면 시트별로 **Disp** 색칠돼 있다:
**🟩 MAKE(차고) / 🟦 BUY(주문) / 🟨 OUTSOURCE(외주) / ⬜ TOOL(공구 1회투자)**.

**발주 순서 = 리드타임 역순으로:**
1. **🟨 외주 먼저(2~4주):** 카본 데크/암 마운트(CNC·워터젯), **페이로드 IF PCB fab+assembly**. → §3.
2. **🟦 핵심 전자(1~3주):** FC(Cube Orange+), GNSS(mosaic-X5), Jetson, ESC×8, 모터×8, RFD900x.
   특히 **mosaic-X5/Jetson/모터**는 품절·리드타임 길다. 일찍.
3. **🟦 페이로드/소물(수일):** Savox 서보×3, Pololu 드라이버, INA226, HX711, 솔레노이드.
4. **🟩 차고 소모품:** 필라멘트(PETG-CF), 와이어/커넥터, 나사, 에폭시, 열전사 용지.

> 💸 현실 체크: 그랜드 토탈 **약 $13k**(공구 $1.9k 포함). 드론이 ~$8k(Jetson·모터·mosaic가 깡패),
> 포렌식 기기 ~$0.9k. "5천원짜리 적 드론 잡는 데 천만원" 소리 또 나오는데, **잡아서 적 기지 좌표
> 뽑으면 본전**이라고 우기면 된다 ㅋㅋ. 가격은 **indicative**, 발주 전 벤더 확인.

## 3. 외주 — 도면/거버 보내기 (차고에서 못 깎는 것)

| 외주 | 보낼 파일 | 어디로 | 팁 |
|------|-----------|--------|-----|
| 카본 데크/랜딩 | `cad/talonet_frame.scad` → STEP/DXF 익스포트 | SendCutSend / 동네 CNC | 3mm CF or G10, 라이트닝홀 포함 |
| 알루 모터마운트/클램프 | STEP | CNC샵 | 6061-T6, 카키 아노다이즈(OS-AN-001) |
| **페이로드 IF PCB** | 거버+BOM+CPL (STM32G474, docs/06 §10) | JLCPCB / PCBWay | fab+**assembly** 같이. rev A 5장 뽑아 여유 |

SCAD → STL/STEP 익스포트:
```bash
openscad -o talonet_frame.stl cad/talonet_frame.scad      # 슬라이서/외주용
# (STEP가 필요하면 FreeCAD로 STL 임포트 후 변환, 또는 SCAD를 CSG로 export)
```
> 카본 가루는 발암성 + 전도성(전자기기 죽임). **외주 맡기는 게 정신건강에 이롭다.**
> 3D프린터(BUY TL-003) 있으면 짐벌요크/스풀/머즐/포렌식 케이스는 **차고에서 PETG-CF로 출력**.

## 4. 드론 "그물매" 빌드

스펙 목표(= 합격 기준, [docs/01](01_드론_스펙.md)): **X8 동축, 휠베이스 1150mm, 공허 6.8kg /
MTOW 14kg, 호버 추력여유 ≥ 2:1.** 이거 못 맞추면 "그물매"가 아니라 "그물 못 매".

### 4.1 프레임 (반나절)
1. 상/하 카본 데크 사이에 **알루 스탠드오프(deck_gap 70mm)** 로 샌드위치.
2. 암 클램프에 카본 튜브(30×26) 꽂고 **에폭시+볼트** 병행, 동축 모터마운트 장착.
3. 랜딩기어 아치 2조. **나사는 전부 Loctite 243**(진동에 풀림 = 공중분해).
4. 토크 드라이버로 규정 토크. "꽉" 말고 **숫자로**.

### 4.2 추진계 (모터/ESC/프롭) — 추력 마진이 생명
1. 모터 8개 동축 장착(상4/하4), ESC 8개 암 안쪽에 타이.
2. **프롭은 아직 달지 마라.** 모터 회전방향만 확인(동축 = 상하 반대방향).
3. ESC ↔ PDB 전원, 신호선 FC 모터레일(DShot600).
4. **추력 검증(프롭 달고, 묶어두고):** 모터 1개 정적추력 측정 → ×8 ≥ **28kgf**(MTOW 14kg의 2:1).
   미달이면 프롭 피치/모터 KV 재검토. (벤치 추력대 or 저울+지그)

### 4.3 항전 스택 (FC/GNSS/컴패니언/전원/C2)
1. **Cube Orange+** 진동 절연 마운트(데크 중앙, 화살표=기수 +X).
2. **mosaic-X5**(주 GNSS, OSNMA) 상단·금속 차폐 멀리, **Here3+**(보조) CAN.
3. **Jetson AGX Orin** + 캐리어, 방열, EO/IR 카메라(CSI/USB3 or RTSP). 9~20V 입력.
4. 전원: 12S → **Mauch HS-200** 파워모듈 → FC POWER1, **Mauch 5.3V BEC ×2 OR-ing** → 항전 버스.
   메인 체결 = **XT90-S 프리차지 후 AS150**(인러시 폭죽 방지).
5. **퓨즈 사이징 = docs/06 §9 그대로**(메인 ANL 150A 등). 와이어 AWG도 표대로. 대충 = 화재.
6. **RFD900x** 한 쌍(드론/지상), TELEM1. MAVLink2.

### 4.4 그물 페이로드 (이게 우리 정체성)
> **단일 진실원천 [`gcs/payload_map.py`](../gcs/payload_map.py):** AIM-PAN=**SERVO9**,
> AIM-TILT=**SERVO10**, CINCH=**SERVO11**, RELEASE=**SERVO12**, NET FIRE=**RELAY0**.
> 차고에서 배선할 때 **이 채널 그대로** FC AUX에 꽂아라. 코드랑 1mm도 안 틀리게.

1. **조준 짐벌**: PETG-CF로 `net_launcher()` 요크/베어링 출력 → **Savox SB-2290SG ×2**(팬/틸트).
   머즐 노즐 장착. **서보 전원은 6V UBEC**(로직 분리). 팬→SERVO9, 틸트→SERVO10.
2. **시닝(입구 조임)**: Pololu 25D 기어모터 + 플랜지 스풀(출력) + **G2 24v13 드라이버** + **INA226**(전류).
   드라이버 PWM→SERVO11. 드로스트링을 스풀에 감고 림 둘레로 라우팅.
3. **윈치(상하 회수)**: Pololu 37D + **G2 24v21** + **로드셀5kg/HX711**(장력).
4. **트리거(발사)**: CO2 NC 솔레노이드 → **IRLB3034 MOSFET** 로우사이드, **아밍 릴레이** 하류,
   **플라이백 다이오드** 필수. 발사 = **RELAY0**(`DO_SET_RELAY`).
5. **그물 제작(차고 장인의 영역)**: 매듭없는 Dyneema/Kevlar 캐스팅 네트(마우스 ~3.5m),
   **드로스트링(퍼스 루프)** + **림 추**(브라스). `capture_net()`은 **발사형** — 머즐에서 사출돼
   펼쳐지게 접어 넣음. 접는 법 = 낙하산 패킹처럼 **꼬임 없이**(꼬이면 공중에서 안 펴짐 = 망신).
6. **모든 코일(솔레노이드/릴레이/모터)**: 플라이백 다이오드. 안 달면 FC가 리셋되며 추락쇼.

### 4.5 배선/크림핑/퓨즈 (지루하지만 여기서 다 갈림)
- 실리콘 와이어 + **제대로 된 크림핑**(납땜만 = 진동에 끊김). 색코드 통일.
- 신호선은 전원선과 분리·트위스트. EMI = 텔레메트리 글리치 = 유령 조종.
- **부팅 안전**: MCU 리셋 시 출력 LOW + **아밍 릴레이 OPEN**(외부 풀다운). docs/06 §10.2 그대로.

### 4.6 펌웨어 (ArduPilot — 진짜 통신 시작)
1. Cube Orange+에 **ArduCopter** 플래시(Mission Planner/QGC). 프레임=**OctoX(동축)**.
2. **서보 함수 매핑(payload_map과 일치):**
   ```
   SERVO9_FUNCTION  = 1   (RCPassThru / 스크립트 제어 — AIM-PAN)
   SERVO10_FUNCTION = 1   (AIM-TILT)
   SERVO11_FUNCTION = 1   (CINCH driver PWM)
   SERVO12_FUNCTION = 1   (RELEASE)
   RELAY1_FUNCTION  = 1   (relay; NET FIRE)
   SERVOn_MIN/MAX/TRIM → §6 캘리브레이션에서 payload_map PWM(1000/1500/2000)과 맞춤
   ```
3. **MANUAL_CONTROL/RC**: 콕핏이 `MANUAL_CONTROL`로 roll/pitch/yaw/throttle 보냄 →
   `RC_OPTIONS`/조종 입력 매핑. 페일세이프(링크두절=RTL→LAND), 지오펜스 설정.
4. **MAVLink2** 활성, TELEM1=RFD900x. (서명은 운용 시 키 프로비저닝, SITL/벤치는 무서명)

## 5. 포렌식 기기 "ForensIQ-1" 빌드 (이건 차고에서 거의 다 됨)

[docs/09](09_Forensic_Appliance_Design.md) 설계 그대로. 드론보다 훨씬 쉽다 — 날지 않으니까.

1. **인클로저**: `cad/forensic_appliance.scad` → STL → **PETG/ASA 출력**(또는 외주). 키락·탬퍼씰 장착.
2. **전장**: **CM4 + 캐리어** → **7" DSI 터치** → **NVMe SSD**(working/case) → **하드웨어
   write-blocker + SD 리더**(증거 카드는 **무조건 이 슬롯**) → **80mm 열전사 프린터**(패널) →
   **UPS HAT + 18650** → **DS3231 RTC**. 전면 LED/버튼(START/EJECT)/키락.
3. **소프트웨어 굽기**:
   ```bash
   pip install -r requirements-forensics.txt     # pymavlink, pyulog, pynmea2, folium, fpdf2
   # 시스템 도구(외부): exiftool, dc3dd, sleuthkit, binwalk (apt)
   python -c "import forensics; print('forensics ready')"
   ```
4. **WORM USB**(증거 내보내기) 후면 포트. 에어갭 기본(이더넷만).

## 6. 소프트웨어 브링업 & 캘리브레이션 (= "코드 = 쇠" 맞추는 단계)

> 여기가 **"찰흙이 진짜와 일치"** 하는 지점. `payload_map.py`의 각도→PWM이 **실제 짐벌 각도**와
> 맞아야 함.

1. **무하드웨어 풀루프(먼저):**
   ```bash
   pip install -r requirements-gcs.txt
   python -m gcs.sim_vehicle                       # 실 MAVLink 차량 에뮬레이터
   python -m gcs --connect udpout:127.0.0.1:14550  # 콕핏 → ARM/조준/발사 명령 실제 전송 확인
   ```
   HUD 좌상단 `LINK: MAVLINK` 녹색 + 텔레메트리 뜨면 SW는 정상.
2. **서보 캘리브레이션(실 짐벌):** 콕핏 `J/L`(팬), `I/K`(틸트)로 슬루 → 짐벌이 **코드 각도만큼**
   실제로 돌게 `SERVO9/10_MIN/MAX/TRIM` 조정. 끝(엔드스톱)에서 기계 간섭 없게 **소프트 리밋 = payload_map(−60..60 / 0..75)**.
3. **실 FC 연결:** `python -m gcs --connect /dev/ttyACM0`(USB) 또는 `udpout:<radio-ip>:14550`.
   EO 카메라: `--video rtsp://<ip>:8554/eo`.
4. 배포·캘리브레이션 상세: [docs/11 CONOPS & 배포](11_CONOPS_and_Deployment.md).

## 7. 벤치 테스트 & 스펙 검증 (합격/불합격 명확히)

> **프롭 빼고, 그물 빼고(dry), 2인.** 통과 못 하면 다음 단계 금지.

| # | 테스트 | 합격 기준 | 어떻게 |
|---|--------|-----------|--------|
| 1 | 전원/부팅 안전 | 부팅 시 모든 페이로드 출력 LOW, 릴레이 OPEN | 멀티미터로 게이트/릴레이 |
| 2 | 텔레메트리 링크 | 콕핏 HUD에 실 자세/GPS/배터리, `LINK LIVE` | `--connect` 실 FC |
| 3 | **그물 조준** | `IJKL`로 팬/틸트 짐벌이 코드 각도대로 회전 | HUD NET 레티클 ↔ 실 짐벌 각도 일치 |
| 4 | 아밍 인터록 | `ARMED` 전엔 FIRE/CINCH 거부, E-Stop 즉시 안전 | 콕핏 + FC 로그 |
| 5 | **발사 트리거(dry)** | `SPACE`→RELAY0 ON 150ms 펄스, 전류 감지 | 그물 빼고 솔레노이드 클릭 |
| 6 | 시닝 장력 | CINCH 모터 권취, 장력/전류 임계서 자동 정지 | INA226/HX711 로그 |
| 7 | 윈치 | 권취/정지, 과장력 자동정지 | 로드셀 |
