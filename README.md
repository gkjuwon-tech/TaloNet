# 🪤 TaloNet — 적 드론 물리적 납치 모선드론 (Counter-UAS Sky Predator)

> "해킹? 스푸핑? 안 함. 그냥 **그물 던져서 잡아옴**."
>
> — TaloNet 창립자, 어느 새벽 3시

---

## TL;DR (바쁜 장군님들을 위한 3줄 요약)

1. **ㅈㄴ 큰 모선 드론**이 하늘을 순찰하다가 적 드론을 발견한다.
2. 그물을 발사해서 **물리적으로 납치**한다. (전자전 아님. 그냥 사로잡음.)
3. 잡은 드론은 **우리 기지로 회수**하거나, "이거 가져오면 ㅈ됨"(자폭형 등)으로 판단되면 **사람 없는 곳에 안전 투하**한다.

끝. 이게 전부다. 그래서 **윤리적으로 ㅈㄴ 깨끗한 방산사업**이다.

---

## 🤔 왜 만드냐 (Why)

요즘 전장(戰場)은 드론 천지다. 정찰 드론, 자폭 드론(FPV), 군집 드론… 하늘에서 윙윙거리는 죄다 위협이다.

기존 대(對)드론 솔루션들은 대충 이렇다:

| 방식 | 문제점 |
|------|--------|
| 🔫 총으로 쏴 떨어뜨림 | 잔해가 사람 위로 떨어짐. 증거인멸. 비쌈. |
| 📡 GPS 스푸핑 / 재밍 | 전파법 위반 소지, 아군 통신도 같이 죽음, 법무팀 야근 |
| 💥 미사일 요격 | 5,000원짜리 드론 잡으려고 5천만원 쓰는 가성비 테러 |
| 🦅 독수리 훈련 (실화) | 독수리 노동조합 결성, 산재보험 이슈 |

**TaloNet의 방식:**

> 그냥 **더 큰 드론**으로 **그물 던져서** 잡아온다. 끝.

- 잔해 안 떨어짐 ✅
- 전파 안 건드림 ✅ (해킹/스푸핑 없음 → 윤리 클린 등급 S)
- 적 드론 **온전히 회수** → 적의 기술 그대로 분석 가능 (보너스)
- 위험하면 사람 없는 곳에 버림 → 민간 피해 0

## 🧩 시스템 구성

```
                  ☁️  TaloNet "그물매(Geulmae)" 모선드론
                 /  \
                /    \   ← 4축 헥사/옥타콥터, 그물런처 탑재
        [그물런처 x2]  [하방 포획 그물]
                |
   📡 내장 VLM (적/아군/자폭 판별) + 멀티센서 융합
                |
        ┌───────┴────────┐
        ▼                ▼
   🏠 기지로 회수      🗑️ 위험물 안전 투하
   (양품: 분석용)      (자폭형: 그물째 낙하)
```

| 구성요소 | 코드네임 | 설명 |
|----------|----------|------|
| 모선 드론 | **그물매 (Geulmae)** | 적 드론보다 ㅈㄴ 큰 헌터-킬러 플랫폼 |
| 포획 그물 | **TaloNet** | 발사형 자기수축 케블라 그물 |
| 포획 운반 | **그물 캐리(Net Carry)** | 잡은 드론을 그물째 담아 매달고 운반 (별도 케이지 X) |
| 두뇌 | **HawkEye-VLM** | 적/아군/위험물 실시간 판별 비전 모델 |
| 위협 분류 | **"ㅈ됨 판정기"** | 가져와도 되는지/버려야 하는지 판단 |

## 🛡️ 방어 코드 (`defense/`)

우리는 **공격은 안 한다.** 적 드론을 해킹/스푸핑/재밍하지 않는다. 대신 **그물매 모선드론 자신**이 당하지 않도록 방어한다. `defense/` 패키지는 전부 **검증된 공개 알고리즘/오픈소스 기반**(내 뇌피셜 아님):

- **GNSS 인증 (anti-spoof):** Galileo OSNMA / TESLA — `defense/gnss/osnma_adapter.py`
- **RAIM (무결성 감시 + 고장위성 배제):** `defense/gnss/raim.py`
- **스푸핑/재밍 일관성 검사** (ArduPilot glitch/EKF gating, C/N0, 클럭점프, 다중위성군): `defense/gnss/spoof_detection.py`
- **MAVLink2 명령 서명** (위조/리플레이 방어): `defense/link/mavlink_signing.py`
- **RF 링크 보안** (HMAC + 안티리플레이 + 재밍/탈취 감시): `defense/link/rf_link_security.py`
- **통합 판정** (TRUST/DEGRADED/DEADRECKON/RTH): `defense/monitor.py`

출처·라이선스: [`defense/README.md`](defense/README.md) · 테스트: `python -m unittest discover -s tests` (47 passing)

## 🔬 포렌식 코드 (`forensics/`)

잡아온 적 드론은 **자기 일기장을 들고 잡혀온** 셈. 해킹/침투가 아니라 **물리적으로 확보된 압류 물품**의 저장장치/로그를 **합법 포렌식**으로 읽는다 (`defense/`가 우리를 지킨다면, `forensics/`는 잡은 걸 분석한다):

- **원본 불가침:** write-blocker 읽기전용 + **SHA-256** 해시 + 원본 봉인 (부팅 금지 → 안티포렌식/자폭 회피)
- **검증 OSS 파서만:** ArduPilot `.bin`(pymavlink) / PX4 `.ulg`(pyulog) / NMEA(pynmea2) / DJI(dji-log-parser) — 뇌피셜 파싱 금지
- **항적·의도 분석:** GPS 항적 → 출발지/타깃 추정(gpxpy/folium), 모든 추정에 신뢰도·근거 첨부
- **무결성/Chain-of-Custody:** append-only **해시체인** 커스터디 로그(`forensics/chain_of_custody.py`, 동작)
- **리포트:** 위협 인텔 리포트(자산화) + CoC 부록

기획: [`docs/08_사후_포렌식.md`](docs/08_사후_포렌식.md) · 모듈: [`forensics/README.md`](forensics/README.md) (현재 **인터페이스/아키텍처 단계**, 어댑터 구현은 다음 스텝)
